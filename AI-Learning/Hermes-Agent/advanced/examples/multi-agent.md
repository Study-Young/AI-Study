# 多代理协作示例

本文演示如何使用 Hermes Agent 的子代理委派功能构建多代理协作系统，让多个 AI 代理协同完成复杂任务。

## 场景：构建一份 AI 行业报告

### 任务描述

用户需要一份全面的 AI 行业分析报告，涵盖市场趋势、技术进展、开源项目和未来预测。

### 实现方案

使用 Hermes Agent 的 `delegate_tasks` 功能，将任务分解给四个专业子代理并行处理。

### 步骤 1：定义任务分解

```yaml
# report-config.yaml
delegation:
  tasks:
    - name: market-analysis
      description: "分析 AI 市场趋势和投融资情况"
      agent_config:
        model: gpt-4o
        temperature: 0.3
      prompt: |
        你是 AI 市场分析专家。请分析当前 AI 市场的以下方面：
        1. 主要 AI 公司的市场表现
        2. 投融资热点领域
        3. 各细分市场规模和增长预估
        请包含数据和引用来源。
    
    - name: tech-progress
      description: "跟踪 AI 技术最新进展"
      agent_config:
        model: claude-sonnet-4-20250514
        temperature: 0.2
      prompt: |
        你是 AI 技术研究专家。请总结以下技术领域的最新进展：
        1. 大语言模型（LLM）
        2. 多模态模型
        3. AI Agent 框架
        4. 模型训练和推理优化
        请关注最近 3 个月内的突破性进展。
    
    - name: open-source
      description: "梳理开源 AI 项目生态"
      agent_config:
        model: gpt-4o-mini
        temperature: 0.4
      prompt: |
        你是开源社区观察员。请梳理当前热门的开源 AI 项目：
        1. 新发布的值得关注的开源项目
        2. 主流框架的版本更新和变化
        3. 社区活跃度和贡献者趋势
        重点关注 GitHub 上的开源项目。
    
    - name: prediction
      description: "AI 行业未来预测"
      agent_config:
        model: gpt-4o
        temperature: 0.7
      prompt: |
        你是 AI 行业战略分析师。基于当前趋势，预测未来 6-12 个月：
        1. 可能的技术突破方向
        2. 行业格局变化
        3. 值得关注的风险和挑战
        请给出具体的预测和时间线。
```

### 步骤 2：执行委派

在聊天中，用户只需说：

```
请为我生成一份 AI 行业分析报告
```

Hermes Agent 自动执行以下操作：

1. **分析任务**：理解用户需求，识别需要四个子报告的复杂报告
2. **分解任务**：将报告任务分解为市场、技术、开源自、预测四个子任务
3. **创建子代理**：并行创建四个子代理，每个配置不同的模型和温度参数
4. **收集结果**：等待所有子代理完成后收集结果
5. **整合报告**：将四份子报告整合为一份完整的综合报告

### 步骤 3：结果整合

主代理使用以下模板整合子代理的结果：

```markdown
# AI 行业分析报告

**生成日期**: {{date}}
**报告摘要**: {{summary}}

---

## 一、市场趋势分析

{{market-analysis}}

## 二、技术进展综述

{{tech-progress}}

## 三、开源生态盘点

{{open-source}}

## 四、未来趋势预测

{{prediction}}

---

## 综合建议

基于以上分析，我们建议：
1. ...
2. ...
3. ...
```

### 步骤 4：输出结果

最终输出是一份完整的 Markdown 报告，包含所有子代理的分析和主代理的综合建议。

## 场景二：研究助手

### 任务描述

用户需要研究一个技术课题，例如："我想了解 RAG（检索增强生成）技术的最新发展"

### 实现方案

```python
# 主代理自动执行以下委派
results = delegate_tasks([
    {
        "name": "paper-review",
        "task": "搜索并总结近期的 RAG 相关研究论文",
        "model": "gpt-4o",
        "context": {"focus": "2025-2026年发表的重要论文"}
    },
    {
        "name": "framework-compare",
        "task": "对比主流的 RAG 框架（LlamaIndex、LangChain、Haystack 等）",
        "model": "gpt-4o"
    },
    {
        "name": "best-practices",
        "task": "整理 RAG 系统的最佳实践和常见优化策略",
        "model": "claude-sonnet-4-20250514"
    },
    {
        "name": "challenges",
        "task": "分析 RAG 技术当前面临的挑战和局限性",
        "model": "gpt-4o-mini"
    }
])

# 整合所有结果为完整研究报告
full_report = integrate_results(results, format="comprehensive")
```

## 场景三：代码审查团队

### 任务描述

用户提交了一个项目的代码库，要求全面审查。

### 实现方案

```
用户请求："审查我的 Flask 项目代码"

主代理自动创建 4 个子代理：

┌─ 子代理 A: 安全审查 ─────────────────────┐
│  模型: gpt-4o                              │
│  检查: SQL注入、XSS、CSRF、认证漏洞等       │
└────────────────────────────────────────────┘

┌─ 子代理 B: 性能审查 ─────────────────────┐
│  模型: gpt-4o                              │
│  检查: N+1查询、缓存策略、异步处理等        │
└────────────────────────────────────────────┘

┌─ 子代理 C: 代码风格审查 ─────────────────┐
│  模型: gpt-4o-mini                         │
│  检查: PEP8 规范、命名约定、注释质量等      │
└────────────────────────────────────────────┘

┌─ 子代理 D: 架构审查 ─────────────────────┐
│  模型: claude-sonnet-4-20250514            │
│  检查: 模块化、可扩展性、设计模式等         │
└────────────────────────────────────────────┘

主代理整合为综合审查报告：
1. 安全漏洞（优先级最高）
2. 性能优化建议
3. 代码质量问题
4. 架构改进建议
```

## 注意事项

### 选择合适的模型

- **分析/推理任务**：使用 `gpt-4o`、`claude-sonnet-4-20250514` 等强大模型
- **简单/总结任务**：使用 `gpt-4o-mini`、`claude-haiku` 等轻量模型
- **创意/生成任务**：使用较高温度（0.7-0.9）
- **分析/精确任务**：使用较低温度（0.1-0.3）

### 资源管理

- **Token 预算**：大型委派任务消耗大量 Token，需注意控制
- **并行数量**：建议不超过 5 个并行子代理，避免 API 限频
- **超时设置**：为每个子代理设置合理的超时时间

### 错误恢复

```yaml
# 建议的容错配置
delegation:
  retry:
    max_retries: 3
    retry_delay: 5
  fallback:
    enabled: true
    model: gpt-4o-mini  # 失败时使用更简单的模型重试
```

## 总结

多代理协作是 Hermes Agent 最强大的功能之一。通过将复杂任务分解给多个专业化子代理，可以实现：

- **并行处理**：大幅缩短复杂任务的完成时间
- **专业化分工**：每个子代理专注于特定领域
- **模型优化**：根据任务复杂度选择合适的模型
- **质量提升**：多角度分析带来更全面的结果
