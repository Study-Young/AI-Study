# 子代理委派

子代理委派（Delegation）是 Hermes Agent 的高级功能，允许主代理创建子代理（Subagent）来并行执行任务。这类似于人类管理者将任务分派给团队成员，极大地提升了处理复杂任务的能力。

## 核心概念

### 委派机制

```
用户请求
     │
     ▼
┌─────────────────┐
│   主代理        │ ◄── 协调者，负责任务分解和结果整合
│   (Main Agent)  │
└────────┬────────┘
         │
    ┌────┴────┬──────────┐
    │         │          │
    ▼         ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│ 子代理1 │ │ 子代理2 │ │ 子代理3 │
│Subagent│ │Subagent│ │Subagent│
└────┬───┘ └────┬───┘ └────┬───┘
     │          │          │
     └──────────┼──────────┘
                ▼
         ┌────────────┐
         │  结果整合   │
         └────────────┘
```

### 委派模式

Hermes Agent 支持两种委派模式：

1. **单次委派（Single Delegate）**：创建一个子代理执行单一任务
2. **批量委派（Batch Delegate）**：创建多个子代理并行执行不同任务

## 函数接口

### delegate_task（单次委派）

创建单个子代理执行特定任务：

```python
result = delegate_task(
    task="搜索最新的 AI 研究论文",
    context={
        "topic": "大语言模型",
        "time_range": "最近一周"
    },
    model="gpt-4o-mini",       # 可选，子代理使用的模型
    temperature=0.3,            # 可选
    max_tokens=4096             # 可选
)
```

### delegate_tasks（批量委派）

同时创建多个子代理并行执行任务：

```python
results = delegate_tasks([
    {
        "task": "搜索 AI 研究论文",
        "context": {"topic": "大语言模型"},
        "model": "gpt-4o-mini"
    },
    {
        "task": "整理开源项目",
        "context": {"topic": "AI Agent"},
        "model": "gpt-4o-mini"
    },
    {
        "task": "收集行业新闻",
        "context": {"topic": "AI 应用"},
        "model": "gpt-4o-mini"
    }
])
```

## 使用场景

### 场景一：并行研究

用户请求："调研最新的 AI Agent 框架"

主代理自动分解任务并委派给多个子代理：

1. **子代理 A**：搜索开源的 AI Agent 框架列表
2. **子代理 B**：收集各框架的功能对比
3. **子代理 C**：查找社区评价和使用案例
4. **结果整合**：主代理将三份报告合并为完整调研文档

### 场景二：代码审查

用户请求："审查我的项目代码"

1. **子代理 A**：检查代码安全问题
2. **子代理 B**：检查代码性能和优化点
3. **子代理 C**：检查代码风格和最佳实践
4. **结果整合**：输出综合审查报告

### 场景三：多语言翻译

用户请求："将这份文档翻译成 5 种语言"

1. **子代理 1**：翻译为英文
2. **子代理 2**：翻译为日文
3. **子代理 3**：翻译为法文
4. **子代理 4**：翻译为德文
5. **子代理 5**：翻译为西班牙文
6. **结果整合**：输出多语言版本

## 配置

```yaml
# ~/.hermes/config.yaml
delegation:
  enabled: true
  
  # 子代理配置
  subagent:
    default_model: gpt-4o-mini     # 子代理默认模型
    max_concurrent: 5              # 最大并行子代理数
    timeout: 120                   # 子代理超时时间（秒）
    
  # 资源限制
  limits:
    max_tokens_per_subagent: 4096  # 子代理最大 Token 数
    max_total_tokens: 32768        # 所有子代理总 Token 数
    max_subagents: 10              # 单次委派最大子代理数
    
  # 重试策略
  retry:
    max_retries: 2                 # 失败重试次数
    retry_delay: 5                 # 重试间隔（秒）
    
  # 结果合并
  merge:
    strategy: sequential           # sequential | parallel
    summarizer_model: gpt-4o       # 结果合并使用的模型
```

## 子代理生命周期

```
创建 (Create)
  │
  ▼
初始化 (Initialize) ── 设置上下文、模型、参数
  │
  ▼
执行 (Execute) ── 子代理独立运行任务
  │
  ├── ✅ 成功 → 返回结果
  │
  └── ❌ 失败 → 重试 → 达到上限 → 返回错误信息
  │
  ▼
完成 (Complete) ── 释放资源，记录日志
```

## 结果合并策略

### 顺序合并（Sequential）

逐个合并子代理的结果，适合需要按顺序处理的情况。

```
结果 A → 合并 A+B → 合并 (A+B)+C → 最终结果
```

### 并行合并（Parallel）

同时合并所有子代理的结果，效率更高。

```
结果 A ─┐
结果 B ─┼─→ 同时合并 → 最终结果
结果 C ─┘
```

## 实践示例

### 示例：生成综合报告

用户："为我生成一份关于 Python 最新趋势的综合报告"

```
主代理分析任务后，创建 3 个子代理：

子代理 1: 任务="搜索 Python 语言新特性"
         上下文={"年份": "2026", "版本": "3.13+"}
         
子代理 2: 任务="搜索 Python 生态工具变化"
         上下文={"领域": "Web开发、数据科学、AI"}
         
子代理 3: 任务="搜索 Python 社区动态"
         上下文={"平台": "PyPI, GitHub, Stack Overflow"}

主代理收到所有结果后，整合为一份结构化的综合报告：
1. 语言特性更新概览
2. 热门工具和库变化
3. 社区趋势分析
4. 未来展望
```

## 监控与调试

### 查看委派状态

```text
/delegation status              # 查看当前委派任务
/delegation list                # 查看历史委派记录
/delegation cancel <id>         # 取消进行中的委派
```

### 日志

委派任务的日志保存在 `~/.hermes/logs/delegation/`：

```
~/.hermes/logs/delegation/
├── 2026-05-15/
│   ├── delegate_001.json       # 委派详情
│   ├── subagent_001.json       # 子代理 1 日志
│   ├── subagent_002.json       # 子代理 2 日志
│   └── result_001.md           # 最终结果
└── ...
```

## 最佳实践

1. **合理划分任务**：将大任务分解为独立的子任务
2. **选择合适的子代理模型**：简单任务用小模型，复杂任务用大模型
3. **设置合理的并行数**：避免过多并行导致 API 限频
4. **处理失败情况**：为委派任务设置重试和错误处理
5. **监控资源消耗**：注意 Token 总量，避免超支
6. **结果验证**：验证子代理返回结果的质量
7. **安全考虑**：子代理继承主代理的安全限制
