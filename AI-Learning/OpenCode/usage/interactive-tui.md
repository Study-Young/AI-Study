# 交互式 TUI 模式

OpenCode 的交互式 TUI（终端用户界面）模式提供了完整的对话式 AI 编程体验。只需在终端中输入 `opencode` 即可启动。

## 启动

```bash
# 在当前项目目录启动
opencode

# 在特定项目目录启动
opencode /path/to/project

# 加载已有的会话
opencode --session <session-id>
```

### 主要区域

1. **标题栏** — 显示版本、当前会话状态和累计费用
2. **对话区域** — 显示 AI 助手的回复和您的历史消息
3. **输入栏** — 输入您的提示词和问题
4. **快捷键栏** — 显示常用快捷键操作

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 发送消息 |
| `Shift+Enter` | 换行 |
| `Ctrl+N` | 新建会话 |
| `Ctrl+S` | 打开设置面板 |
| `Ctrl+O` | 打开会话列表 |
| `Ctrl+E` | 导出当前会话 |
| `Ctrl+L` | 清屏 |
| `Ctrl+C` | 退出 TUI |
| `Ctrl+Z` | 暂停当前生成 |
| `Tab` | 在输入框和命令面板间切换 |
| `Esc` | 关闭面板 / 取消操作 |
| `↑` / `↓` | 浏览历史消息或选择项 |
| `Ctrl+F` | 搜索对话内容 |

## 核心功能

### 多轮对话

TUI 模式的核心优势是保持上下文连续性。您可以：

```text
您: 帮我创建一个 Express.js 路由文件

AI: [创建了 routes/users.js]

您: 现在为这个路由添加 JWT 验证中间件

AI: [在 routes/users.js 中添加了 auth 中间件]

您: 很好，再帮我写对应的测试文件

AI: [在 tests/ 下创建了 users.test.js]
```

OpenCode 会记住之前的对话和生成的文件，在后续回复中保持上下文一致。

### 会话管理

```bash
# 在 TUI 中按 Ctrl+O 打开会话列表

# 会话列表会显示：
# ┌─────────────────────────────────────┐
# │ 📋 会话列表                          │
# │─────────────────────────────────────│
# │ > 当前会话 (active)                  │
# │   2026-05-15 10:30 | 12 条消息       │
# │─────────────────────────────────────│
# │   auth-api-work                      │
# │   2026-05-14 15:20 | 8 条消息        │
# │─────────────────────────────────────│
# │   database-refactor                  │
# │   2026-05-13 09:00 | 24 条消息       │
# │─────────────────────────────────────│
# │ [Enter] 加载  [Delete] 删除  [Esc] 取消│
```

- 会话自动保存，无需手动操作
- 可随时切换回之前的会话
- 会话包含完整的对话历史和上下文

### 文件操作

OpenCode TUI 模式允许直接在对话中操作文件：

- **创建文件**: "创建一个文件 src/utils/helpers.ts"
- **编辑文件**: "修改 src/config.ts 中的数据库连接字符串"
- **查看文件**: "显示 src/main.ts 的内容"
- **删除文件**: "删除不再需要的 test/old-test.js"

AI 会在回复中直接嵌入文件变更，并以 diff 格式展示：

```diff
+ // 新增文件: src/utils/logger.ts
+ export function log(message: string): void {
+   console.log(`[${new Date().toISOString()}] ${message}`);
+ }

- // 移除旧的 console.log
- console.log('started');
+ log('Application started');
```

### 成本追踪

TUI 模式实时显示当前会话的 API 费用：

- 标题栏显示本次会话累计费用
- 每条回复显示本次调用的 token 使用量和费用
- 可设置预算上限，防止意外超支

## 使用技巧

### 高效提示词

```text
❌ 模糊: "帮我写代码"
✅ 具体: "创建一个 Express.js 中间件函数，用于验证 JWT token，包含错误处理"
```

### 利用项目上下文

```text
"查看 src/services/ 目录下的所有服务类，分析它们之间的依赖关系，并给出优化建议"
```

### 迭代式开发

```text
轮次 1: "创建一个 React 用户列表组件"
轮次 2: "添加分页功能"
轮次 3: "添加搜索和筛选"
轮次 4: "为这个组件添加单元测试"
```

### 同时处理多个文件

```text
"重构 src/api/ 目录下的所有路由文件，将公共逻辑提取到中间件中"
```

## 故障排除

### 界面显示异常

```bash
# 确保终端支持 256 色
export TERM=xterm-256color

# 或者在 settings 中切换主题
# 按 Ctrl+S > 选择主题 > 切换
```

### 响应中断

- 按 `Ctrl+Z` 暂停当前生成
- 按 `Ctrl+N` 强制开始新会话
- 检查网络连接和 API 密钥状态

### 性能问题

对于大型项目，OpenCode 会自动优化上下文窗口。如果感觉响应变慢：
- 使用 `Ctrl+N` 新建一个干净的会话
- 缩小上下文范围："只看 src/core/ 目录"
