## 需求 2：AI 资讯文件格式优化

### 当前格式（2026-05-15.md）
```
## 热门论文
- [标题](链接) - 一句话摘要

## 行业动态
- 要点
```

### 目标格式
```
## 热门论文
- [[时间戳] [置信度]] [标题](链接) — 摘要
```

**示例：**
```
## 热门论文
- [2026-05-14 | ★★★★☆] [GPT-5 早期预览曝光](https://...) — OpenAI 在内部测试中展示了...
- [2026-05-13 | ★★★☆☆] [Claude Enterprise 计划](https://...) — Anthropic 面向企业的订阅方案...

## 行业动态
- [2026-05-14 | ★★★★★] 欧盟 AI 法案正式生效，对高风险 AI 系统实施严格监管
- [2026-05-12 | ★★★☆☆] Meta 开源 LLaMA-4 405B 模型权重

## 开源项目
- [2026-05-14 | ★★★★☆] [opencode](https://...) — 发布 v0.x，新增...
```

### 置信度标准
| 星级 | 含义 | 适用场景 |
|------|------|---------|
| ★★★★★ | 官方确认、已发布/已生效 | 官方公告、正式发布 |
| ★★★★☆ | 可靠来源、基本确认 | 主流媒体报道、知名机构消息 |
| ★★★☆☆ | 一般来源、有待验证 | 博客、社区讨论、未确认传闻 |
| ★★☆☆☆ | 传闻阶段、需谨慎 | 小道消息、未证实信息 |
| ★☆☆☆☆ | 存疑、需进一步核实 | 无法确认来源的信息 |

### 执行步骤
1. 读取当前示例文件 `AI-News/news/2026-05-15.md`
2. 按新格式重写该文件，为每条资讯补充时间戳和置信度
3. 同步更新 `AI-News/README.md` 中关于输出格式的说明

## 需求 3：每天早上 9 点定时采集 AI 资讯

### 技术方案：使用 Hermes cronjob 工具

利用 Hermes 自带的 cron 系统，创建定时任务。

**采集方案：**
1. 使用 `web_search` 查询多个主流 AI 信息源
2. 将搜索结果提炼总结为结构化文档
3. 按新格式保存到 `AI-News/news/YYYY-MM-DD.md`
4. 自动提交到 GitHub

**工作流设计：**
```
每天早上 9:00
  ↓
Hermes cron 触发
  ↓
web_search 采集多个 AI 源
  ↓
AI 提炼总结 → 生成 Markdown 文件
  ↓
git add + git commit + git push
```

**涉及的信息源：**
- 论文：ArXiv 当日热门论文
- 行业动态：TechCrunch AI / The Verge AI
- 开源：GitHub Trending
- 大厂：OpenAI / Anthropic / Google AI Blog
- 中文：机器之心 / 量子位（可选）

### 疑问/待确认
- ❓ 是否需要同时推送到 GitHub？还是仅本地保存？
- ❓ 采集频率：每天 9:00 采集的是前一天的全量资讯，是否需要调整为早晚各一次？
- ❓ 资讯内容偏好：偏向论文、行业动态还是开源项目？
